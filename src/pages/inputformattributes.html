• Atributos do Elemento <input> em <form>

    O Atributo form

    O atributo form especifica a qual formulário o elemento <input> pertence.

    O valor deste atributo deve ser igual ao atributo id do elemento <form> ao qual pertence.

    Exemplo

    Um campo de entrada localizado fora do formulário HTML (mas ainda parte do formulário):

                        <form action="/action_page.php" id="form1">
        <label for="fname">Primeiro nome:</label>
        <input type="text" id="fname" name="fname"/>
        <br><br>
        <input type="submit" value="Enviar"/>
     </form>

     <label for="lname">Último nome:</label>
     <input type="text" id="lname" name="lname" form="form1"/>

    Resultado



    O Atributo formaction

    O atributo formaction especifica a URL do arquivo que processará os dados quando o formulário for enviado.

    Nota: Este atributo substitui o atributo action do elemento <form>.

    Exemplo

    Um formulário HTML com dois botões de envio, com ações diferentes:

                        <form action="/action_page.php">
        <label for="fname">Primeiro nome:</label>
        <input type="text" id="fname" name="fname"/> <br><br>
        <label for="lname">Último nome:</label>
        <input type="text" id="lname" name="lname"/> <br><br>
        <input type="submit" value="Enviar"/>
        <input type="submit" formaction="/action_page2.php" value="Enviar como Admin"/>
     </form>

    Resultado





    O Atributo formenctype

    O atributo formenctype especifica como os dados do formulário devem ser codificados ao serem enviados (apenas para formulários com método post).

    Nota: Este atributo substitui o atributo enctype do elemento <form>.

    Exemplo

    Um formulário com dois botões de envio. O primeiro envia os dados do formulário com a codificação padrão, e o segundo envia os dados do formulário codificados como "multipart/form-data":

                        <form action="/action_page_binary.asp" method="post">
        <label for="fname">Primeiro nome:</label>
        <input type="text" id="fname" name="fname"/> <br><br>
        <input type="submit" value="Enviar"/>
        <input type="submit" formenctype="multipart/form-data" value="Enviar como Multipart/form-data"/>
     </form>

    Resultado



    O Atributo formmethod

    O atributo formmethod define o método HTTP para enviar os dados do formulário para a URL de ação.

    Nota: Este atributo substitui o atributo method do elemento <form>.

    Exemplo

    Um formulário com dois botões de envio. O primeiro usa o método "get" e o segundo usa o método "post":

                        <form action="/action_page.php" method="get">
        <label for="fname">Primeiro nome:</label>
        <input type="text" id="fname" name="fname"/> <br><br>
        <label for="lname">Último nome:</label>
        <input type="text" id="lname" name="lname"/> <br><br>
        <input type="submit" value="Enviar usando GET"/>
        <input type="submit" formmethod="post" value="Enviar usando POST"/>
     </form>

    Resultado





    O Atributo formtarget

    O atributo formtarget especifica um nome ou palavra-chave que indica onde exibir a resposta recebida após enviar o formulário.

    Nota: Este atributo substitui o atributo target do elemento <form>.

    Exemplo

    Um formulário com dois botões de envio, com diferentes destinos:

                        <form action="/action_page.php">
        <label for="fname">Primeiro nome:</label>
        <input type="text" id="fname" name="fname"/> <br><br>
        <label for="lname">Último nome:</label>
        <input type="text" id="lname" name="lname"/> <br><br>
        <input type="submit" value="Enviar"/>
        <input type="submit" formtarget="_blank" value="Enviar para uma nova aba/janela"/>
     </form>

    Resultado





    O Atributo formnovalidate

    O atributo formnovalidate especifica que um elemento <input> não deve ser validado ao ser enviado.

    Nota: Este atributo substitui o atributo novalidate do elemento <form>.

    Exemplo

    Um formulário com dois botões de envio, com e sem validação:

                        <form action="/action_page.php">
        <label for="email">Digite seu e-mail:</label>
        <input type="email" id="email" name="email"/> <br><br>
        <input type="submit" value="Enviar"/>
        <input type="submit" formnovalidate="formnovalidate" value="Enviar sem validação"/>
     </form>

    Resultado



    O Atributo novalidate

    O atributo <form> chamado novalidate especifica que todos os dados do formulário não devem ser validados ao serem enviados.

    Exemplo

                        <form action="/action_page.php" novalidate>
        <label for="email">Digite seu e-mail:</label>
        <input type="email" id="email" name="email"/> <br><br>
        <input type="submit" value="Enviar"/>
     </form>

    Resultado
