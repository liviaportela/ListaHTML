• Guia de Estilo HTML

    Um código HTML consistente, limpo e organizado torna mais fácil para outras pessoas ler e entender seu código.

    Boas Práticas e Dicas para Criar um Bom Código HTML

    Sempre Declare o Tipo de Documento

    Sempre declare o tipo de documento como a primeira linha do seu documento.

    O tipo de documento correto para HTML é:

                        <!DOCTYPE html>

    Use Nomes de Elementos em Minúsculas

    O HTML permite misturar letras maiúsculas e minúsculas nos nomes dos elementos.

    No entanto, recomendamos usar nomes de elementos em minúsculas, porque:

    • Misturar maiúsculas e minúsculas não fica bom
    • Os desenvolvedores normalmente usam minúsculas
    • Minúsculas parecem mais limpas
    • Minúsculas são mais fáceis de escrever

    Exemplo de bom código:

                        <body>
        <p>Este é um parágrafo.</p>
     </body>

    Resultado

    Este é um parágrafo.

    Exemplo de mau código:

                        <BODY>
        <P>Este é um parágrafo.</P>
     </BODY>

    Resultado

    Este é um parágrafo.

    Feche Todos os Elementos HTML

    No HTML, você não precisa fechar todos os elementos (por exemplo, o elemento <p>).

    No entanto, recomendamos fortemente fechar todos os elementos HTML, como este exemplo de bom código: 

                        <section>
        <p>Este é um parágrafo.</p>
        <p>Este é um parágrafo.</p>
     </section>

    Resultado

    Este é um parágrafo.

    Este é um parágrafo.

                        <section>
        <p>Este é um parágrafo.
        <p>Este é um parágrafo.
     </section>

    Resultado

    Este é um parágrafo.

    Este é um parágrafo.

    Use Nomes de Atributo em Minúsculas

    O HTML permite misturar letras maiúsculas e minúsculas nos nomes dos atributos.

    No entanto, recomendamos usar nomes de atributos em minúsculas, porque:

    • Misturar maiúsculas e minúsculas não fica bom
    • Os desenvolvedores normalmente usam minúsculas
    • Minúsculas parecem mais limpas
    • Minúsculas são mais fáceis de escrever

    Exemplo de bom código:

                        <a href="https://www.w3schools.com/html/">Visite nosso tutorial de HTML</a>

    Resultado

    Visite nosso tutorial de HTML

    Exemplo de mau código:

                        <a HREF="https://www.w3schools.com/html/">Visite nosso tutorial de HTML</a>

    Resultado

    Visite nosso tutorial de HTML

    Sempre Cite os Valores dos Atributos

    O HTML permite valores de atributos sem aspas.

    No entanto, recomendamos citar os valores dos atributos, porque:

    • Os desenvolvedores normalmente citam valores de atributos
    • Valores entre aspas são mais fáceis de ler
    • Você DEVE usar aspas se o valor contiver espaços

    Exemplo de bom código:

                        <table class="striped">

    Exemplo de mau código:

                        <table class=striped>

    Exemplo de código muito ruim:

                        <table class=table striped>

    Especificar o Atributo alt, Largura e Altura para Imagens

    Sempre especifique o atributo alt para imagens. Este atributo é importante se a imagem não puder ser exibida.

    Também é recomendável definir a largura e a altura das imagens para evitar cintilação, pois o navegador pode reservar espaço para a imagem antes do carregamento.

    Exemplo de bom código:

                        <img src="html5.gif" alt="HTML5" style="width: 128px; height: 128px">

    Resultado

    Exemplo de mau código:

                        <img src="../media/Dog.png">

    Resultado

    Espaços e Sinais de Igualdade

    HTML permite espaços em torno de sinais de igual. Mas sem espaço é mais fácil de ler e agrupa melhor as entidades.

    Exemplo de bom código:

                        <link rel="stylesheet" href="styles.css">

    Exemplo de mau código:

                        <link  rel = "stylesheet" href = "styles.css">

    Evitar Linhas de Código Muito Longas

    Quando usar um editor de HTML, não é conveniente rolar para a direita e para a esquerda para ler o código HTML. Tente evitar linhas de código muito longas.

    Linhas em Branco e Indentação

    Não adicione linhas em branco, espaços ou indentação sem um motivo.

    Para melhorar a legibilidade, adicione linhas em branco para separar grandes blocos ou blocos lógicos de código.

    Para melhorar a legibilidade, adicione duas espaços de indentação. Não use a tecla de tabulação.

    Exemplo de bom código

                        <body>

        <h1>Cidades Famosas</h1>

        <h2>Tóquio</h2>
        <p>Tóquio é a capital do Japão, o centro da Grande Tóquio e a área metropolitana mais populosa do mundo.</p>

        <h2>Londres</h2>
        <p>Londres é a capital da Inglaterra. É a cidade mais populosa do Reino Unido.</p>

        <h2>Paris</h2>
        <p>Paris é a capital da França. A área de Paris é um dos maiores centros populacionais da Europa.</p>
     </body>

    Exemplo de mal código

                        <body>
        <h1>Cidades Famosas</h1>
        <h2>Tóquio</h2>
        <p>Tóquio é a capital do Japão, o centro da Grande Tóquio, e a área metropolitana mais populosa do mundo.</p>
        <h2>Londres</h2>
        <p>Londres é a capital da Inglaterra. É a cidade mais populosa do Reino Unido.</p>
        <h2>Paris</h2>
        <p>Paris é a capital da França. A área de Paris é um dos maiores centros populacionais da Europa.</p>
    </body>

    O Elemento <title> Nunca Deve Ser Omitido

    O elemento <title> é obrigatório em HTML.

    O conteúdo do título da página é muito importante para a otimização de mecanismos de busca (SEO)! O título da página é usado por algoritmos de mecanismos de busca para decidir a ordem ao listar páginas nos resultados de busca.

                            <title>Guia de Estilo HTML e Convenções de Codificação</title>

    Posso Omitir <html> e <body>?

    Uma página HTML será validada sem as tags <html> e <body>:

         <!DOCTYPE html>
     <html>
        <title>Título da Página</title>
     <body>
        <h1>Este é um cabeçalho</h1>
        <p>Este é um parágrafo.</p>
     </body>
     </html>

    No entanto, recomendamos sempre adicionar as tags <html> e <body>!

    Omitir <body> pode causar erros em navegadores mais antigos.

    Omitir <html> e <body> também pode quebrar software de DOM e XML.

    Posso Omitir <head>?

    A tag <head> também pode ser omitida.

    Os navegadores irão adicionar todos os elementos antes de <body> a um elemento <head> padrão. 

                        <!DOCTYPE html>
     <html>
     <title>Título da Página</title>
     <body>
        <h1>Este é um cabeçalho</h1>
        <p>Este é um parágrafo.</p>
     </body>
     </html>

    Devo Fechar Elementos HTML Vazios?

    No HTML, é opcional fechar elementos vazios.

    Permitido:

                        <meta charset="utf-8">

    Também Permitido:

                        <meta charset="utf-8" />

    Se você espera que software XML/XHTML acesse sua página, mantenha a barra de fechamento (/), pois ela é necessária no XML e XHTML.

    Adicione o Atributo lang

    Sempre inclua o atributo lang dentro da tag <html>, para declarar o idioma da página da web. Isso ajuda os mecanismos de busca e navegadores.

                        <!DOCTYPE html>
     <html lang="en-us">
     <head>
        <title>Título da Página</title>
     </head>
     <body>
        <h1>Este é um cabeçalho</h1>
        <p>Este é um parágrafo.</p>
     </body>
     </html>

    Meta Dados

    Para garantir a interpretação correta e indexação adequada pelos mecanismos de busca, tanto o idioma quanto a codificação de caracteres (<meta charset="charset">) devem ser definidos o mais cedo possível em um documento HTML.

                        <!DOCTYPE html>
     <html lang="en-us">
     <head>
        <meta charset="UTF-8">
        <title>Título da Página</title>
     </head>

    Configurando o Viewport

    O viewport é a área visível de uma página da web para o usuário. Varia conforme o dispositivo - será menor em um celular do que em uma tela de computador.

    Você deve incluir o seguinte elemento <meta> em todas as suas páginas da web:

                        <meta name="viewport" content="width=device-width, initial-scale=1.0">

    Isso dá ao navegador instruções sobre como controlar as dimensões e a escala da página.

    O parâmetro width=device-width define a largura da página para seguir a largura da tela do dispositivo (que variará conforme o dispositivo).

    O parâmetro initial-scale=1.0 define o nível de zoom inicial quando a página é carregada pelo navegador.

    Comentários em HTML

    Comentários curtos devem ser escritos em uma única linha, como este:

                        <!-- Este é um comentário -->

    Comentários que se estendem por mais de uma linha devem ser escritos assim:

                        <!--
        Este é um exemplo de comentário longo. Este é um exemplo de comentário longo.
        Este é um exemplo de comentário longo. Este é um exemplo de comentário longo.
    -->

    Comentários longos são mais fáceis de ler se forem recuados com dois espaços.

    Uso de Folhas de Estilo

    Use sintaxe simples para vincular folhas de estilo (o atributo type não é necessário): 

                        <link rel="stylesheet" href="styles.css">

    Regras de CSS curtas podem ser escritas de forma compacta, como este exemplo:

                        p.intro {
        font-family: Verdana;
        font-size: 16em;
     }

    Regras de CSS mais longas devem ser escritas em várias linhas:

                        body {
        background-color: lightgrey;
        font-family: "Arial Black", Helvetica, sans-serif;
        font-size: 16em;
        color: black;
    }

    Carregando JavaScript em HTML

    Use sintaxe simples para carregar scripts externos (o atributo type não é necessário): 

                        <script src="myscript.js"></script>

    Acessando Elementos HTML com JavaScript

    Usar código HTML "desarrumado" pode resultar em erros de JavaScript.

    Essas duas instruções JavaScript produzem resultados diferentes:

    Exemplo

                        getElementById("Demo").innerHTML = "Hello";
     getElementById("demo").innerHTML = "Hello";

    Arquivo de Nomes em Minúsculas

    Alguns servidores web (Apache, Unix) são sensíveis a maiúsculas/minúsculas em nomes de arquivos: "london.jpg" não pode ser acessado como "London.jpg".

    Outros servidores web (Microsoft, IIS) não são sensíveis a maiúsculas/minúsculas: "london.jpg" pode ser acessado como "London.jpg".

    Se você usa uma mistura de maiúsculas e minúsculas, deve estar ciente disso.

    Para evitar esses problemas, use sempre nomes de arquivos em minúsculas!

    Extensões de Arquivo

    Os arquivos HTML devem ter uma extensão .html (.htm é permitido).

    Os arquivos CSS devem ter uma extensão .css.

    Os arquivos JavaScript devem ter uma extensão .js.

    Diferenças Entre .htm e .html?

    Não há diferença entre as extensões de arquivo .htm e .html!

    Ambas serão tratadas como HTML por qualquer navegador e servidor web.

    Nomes de Arquivo Padrão

    Quando um URL não especifica um nome de arquivo no final (como "https://www.w3schools.com/"), o servidor apenas adiciona um nome de arquivo padrão, como "index.html", "index.htm", "default.html", ou "default.htm".

    Se seu servidor for configurado apenas com "index.html" como nome de arquivo padrão, seu arquivo deve ser chamado "index.html", e não "default.html".

    No entanto, os servidores podem ser configurados com mais de um nome de arquivo padrão; normalmente, você pode configurar quantos nomes de arquivo padrão quiser.
