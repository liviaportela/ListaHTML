• HTML URL Encode

    Uma URL é outro termo para endereço da web. Pode ser composta por palavras (por exemplo, w3schools.com), ou um endereço de Protocolo da Internet (IP) (por exemplo, 192.68.20.50).

    A maioria das pessoas digita o nome ao navegar, porque nomes são mais fáceis de lembrar do que números.

    Esquema de URL

    Os navegadores da web solicitam páginas dos servidores da web usando uma URL. Uma URL é usada para endereçar um documento (ou outros dados) na web.

    Um endereço da web como https://www.w3schools.com/html/default.asp segue estas regras de sintaxe:

                        <p><strong>scheme://prefix.domain:port/path/filename</strong></p>

    Resultado

    scheme://prefix.domain:port/path/filename

    Explicação:

    scheme - define o tipo de serviço na Internet (os mais comuns são http ou https)

    prefix - define um prefixo de domínio (o padrão para http é www)

    domain - define o nome do domínio da Internet (como w3schools.com)

    port - define o número da porta no host (o padrão para http é 80)

    path - define um caminho no servidor (se omitido, é o diretório raiz do site)

    filename - define o nome de um documento ou recurso

    • Esquemas Comuns de URL

    Esquema Significado Uso
    http HyperText Transfer Protocol Páginas da web comuns, não criptografadas
    https Secure HyperText Transfer Protocol Páginas da web seguras, criptografadas
    ftp File Transfer Protocol Download ou upload de arquivos
    file Arquivo local Um arquivo no seu computador

    Codificação da URL

    URLs só podem ser enviadas pela Internet usando o conjunto de caracteres ASCII. Se uma URL contém caracteres fora do conjunto ASCII, ela deve ser convertida.

    O URL encoding converte caracteres não-ASCII em um formato que pode ser transmitido pela Internet. Substitui caracteres não-ASCII por "%" seguido de dígitos hexadecimais.

    URLs não podem conter espaços. Normalmente, o URL encoding substitui um espaço por um sinal de mais (+), ou %20.
