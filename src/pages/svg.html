• HTML SVG

    SVG significa Scalable Vector Graphics e é usado para definir gráficos vetoriais para a Web.

    Os gráficos SVG são escaláveis e não perdem qualidade ao serem ampliados ou redimensionados.

    Características do SVG

    SVG define gráficos em formato XML.

    Cada elemento e atributo em arquivos SVG podem ser animados.

    SVG é uma recomendação W3C.

    SVG se integra com outros padrões como CSS, DOM, XSL e JavaScript.

    O Elemento <svg>

    O elemento HTML <svg> é um contêiner para gráficos SVG (Scalable Vector Graphics).

    SVG é uma tecnologia que permite desenhar gráficos vetoriais diretamente nas páginas HTML. Ela oferece várias formas de criar gráficos complexos, incluindo caminhos, retângulos, círculos, polígonos, texto e muito mais.

    Exemplos de Elementos SVG

    Circulo SVG

                        <svg width="100" height="100">
        <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
     </svg>

    Resultado

    Retângulo SVG

                        <svg width="220" height="120">
         <rect x="10" y="10" width="200" height="100" stroke="red" stroke-width="6" fill="blue" /> 
     </svg>

    Resultado

    Retângulo SVG com Opacidade e Cantos Arredondados

                            <svg width="220" height="180">
         <rect x="50" y="20" rx="20" ry="20" width="150" height="150" style="fill: red; stroke: black; stroke-width: 5; opacity: 0.5" /> 
     </svg>

    Resultado

    Estrela SVG

                        <svg width="220" height="200">
         <polygon points="100,10 40,198 190,78 10,78 160,198" style="fill: lime; stroke: purple; stroke-width: 5; fill-rule: evenodd;" /> 
     </svg>

    Resultado

    Elipse e Texto com Gradiente SVG

                        <svg height="130" width="220">
        <defs>
            <linearGradient id="grad1">
            <stop offset="0%" stop-color="yellow" />
            <stop offset="100%" stop-color="red" />
            </linearGradient>
        </defs>

        <ellipse cx="100" cy="70" rx="85" ry="55" fill="url(#grad1)" />

        <text fill="#ffffff" font-size="45" font-family="Verdana" x="50" y="86">SVG</text>
        Desculpe, seu navegador não suporta SVG inline.
     </svg>

    Resultado

    Diferenças Entre SVG e Canvas

    SVG é uma linguagem para descrever gráficos 2D em XML, enquanto Canvas desenha gráficos 2D na hora usando JavaScript.

    Características do SVG

    • SVG é baseado em XML, permitindo anexar manipuladores de eventos JavaScript.
    • SVG lembra cada forma desenhada como um objeto. Se atributos de um objeto SVG forem alterados, o navegador pode re-renderizar automaticamente a forma.

    Características do Canvas

    • Canvas é renderizado pixel a pixel.
    • Se a posição de um gráfico Canvas for alterada, a cena inteira precisa ser redesenhada, incluindo os objetos cobertos pelo gráfico.

    Comparação entre SVG e Canvas

    SVG Canvas
    Independente da resolução Dependente da resolução
    Suporte a manipuladores de eventos Sem suporte para manipuladores de eventos
    Boa capacidade de renderização de texto Baixa capacidade de renderização de texto
    Lento para renderizações complexas Bem adaptado para jogos gráficos intensivos
    Não é adequado para aplicativos de jogos Pode salvar imagens como .png ou .jpg
