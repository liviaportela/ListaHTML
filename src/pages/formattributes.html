• Atributos do Elemento <form>

    Os atributos de formulário controlam seu comportamento, como método de envio e destino.

    O Atributo Action

    O atributo action define a ação a ser realizada quando o formulário é enviado.

    Normalmente, os dados do formulário são enviados para um arquivo no servidor quando o usuário clica no botão de envio.

    No exemplo abaixo, os dados do formulário são enviados para um arquivo chamado "action_page.php". Este arquivo contém um script do lado do servidor que processa os dados do formulário:

    Exemplo

                            <form action="/action_page.php">
        <label for="fname">Nome:</label><br>
        <input type="text"id="fname" name="fname" value="John" /><br><br>
        <label for="lname">Sobrenome:</label><br>
        <input type="text" id="lname" name="lname" value="Doe" /><br><br>
        <input type="submit" value="Enviar" />
     </form>

    Resultado







    Dica: Se o atributo action for omitido, a ação será definida para a página atual.

    O Atributo Target

    O atributo target especifica onde exibir a resposta recebida após o envio do formulário.

    O atributo target pode ter um dos seguintes valores:

    Valor Descrição
    _blank A resposta é exibida em uma nova janela ou aba
    _self A resposta é exibida na janela atual
    _parent A resposta é exibida no frame pai
    _top A resposta é exibida no corpo completo da janela

    O valor padrão é _self, o que significa que a resposta será aberta na janela atual.

    Exemplo:

    Aqui, o resultado enviado será aberto em uma nova aba do navegador:

                        <form action="/action_page.php" target="_blank">
        <input type="submit" value="Enviar"/>
     </form>

    Resultado

    O Atributo Method

    O atributo method especifica o método HTTP a ser usado ao enviar dados do formulário.

    Os dados do formulário podem ser enviados como variáveis de URL (com method="get") ou como uma transação HTTP post (com method="post").

    O método HTTP padrão ao enviar dados do formulário é GET.

    Exemplo com GET:

                        <form action="/action_page.php" method="get">
        <input type="submit" value="Enviar"/>
     </form>

    Resultado

    Exemplo com POST:

                        <form action="/action_page.php" method="post">
        <input type="submit" value="Enviar"/>
     </form>

    Resultado

    Notas sobre GET:

    • Anexa os dados do formulário à URL, em pares de nome/valor
    • NUNCA use GET para enviar dados sensíveis! (os dados enviados pelo formulário ficam visíveis na URL)
    • O comprimento de uma URL é limitado (2048 caracteres)
    • Útil para envios de formulário em que um usuário deseja adicionar aos favoritos
    • GET é bom para dados não seguros, como strings de consulta no Google

    Notas sobre POST:

    • Anexa os dados do formulário no corpo da solicitação HTTP (os dados enviados pelo formulário não são mostrados na URL)
    • POST não tem limitações de tamanho e pode ser usado para enviar grandes quantidades de dados
    • Envios de formulário com POST não podem ser adicionados aos favoritos

    Dica: Sempre use POST se os dados do formulário contiverem informações sensíveis ou pessoais!

    O Atributo Autocomplete

    O atributo autocomplete especifica se um formulário deve ter preenchimento automático ligado ou desligado.

    Quando o autocomplete está ativado, o navegador completa automaticamente os valores com base em valores que o usuário inseriu anteriormente.

    Exemplo com autocomplete ligado:

                        <form action="/action_page.php" autocomplete="on">
        <input type="submit" value="Enviar"/>
     </form>

    Resultado

    O Atributo Novalidate

    O atributo novalidate é um atributo booleano.

    Quando presente, ele especifica que os dados do formulário (entrada) não devem ser validados ao serem enviados.

    Exemplo com novalidate:

                        <form action="/action_page.php" novalidate>
        <input type="submit" value="Enviar"/>
     </form>

    Resultado

    Lista de Todos os Atributos <form>

    Atributo Descrição
    accept-charset Especifica as codificações de caracteres usadas para envio do formulário
    action Especifica para onde enviar os dados do formulário ao enviar o formulário
    autocomplete Especifica se um formulário deve ter preenchimento automático ligado ou desligado
    enctype Especifica como os dados do formulário devem ser codificados ao enviá-los ao servidor (apenas para method="post")
    method Especifica o método HTTP a ser usado ao enviar dados do formulário
    name Especifica o nome do formulário
    novalidate Especifica que o formulário não deve ser validado ao ser enviado
    rel Especifica a relação entre um recurso vinculado e o documento atual
    target Especifica onde exibir a resposta recebida após o envio do formulário
